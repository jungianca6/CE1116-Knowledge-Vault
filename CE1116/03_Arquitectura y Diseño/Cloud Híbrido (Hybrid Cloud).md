## 📚 Idea/Concepto 

La nube híbrida combina entornos de nube pública, nube privada e infraestructura local (on-premise) en una única infraestructura de TI orquestada e interoperable, donde los datos y las aplicaciones pueden moverse fluidamente entre entornos. No basta con tener ambos; se requiere integración activa mediante conectividad robusta (VPN, conexiones dedicadas como AWS Direct Connect o Azure ExpressRoute). Una ventaja fundamental es la portabilidad de cargas de trabajo ("build once, deploy anywhere"), a menudo lograda mediante contenedores (Kubernetes/OpenShift) que abstraen la infraestructura subyacente. Este modelo permite elegir dónde ejecutar cada carga según necesidades de coste, escalabilidad o cumplimiento normativo (ej. requisitos de soberanía de datos como GDPR, que obligan a mantener ciertos datos on-premise). También aumenta la superficie de ataque, por lo que la seguridad debe gestionarse de forma coherente entre entornos.
