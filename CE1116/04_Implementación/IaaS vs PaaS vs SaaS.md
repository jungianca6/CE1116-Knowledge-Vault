## 📚 Idea/Concepto 

IaaS (Infraestructura como Servicio): acceso bajo demanda a cómputo, almacenamiento y redes. El cliente controla sistemas operativos, middleware y aplicaciones, pero NO la infraestructura física subyacente. Responsabilidades del cliente: aplicar parches de seguridad al SO, gestionar el runtime y los datos.

PaaS (Plataforma como Servicio): proporciona una plataforma lista para desarrollar y desplegar aplicaciones. El proveedor gestiona hardware, sistema operativo, runtime y middleware. El desarrollador solo se preocupa por código y datos. No hay interacción directa con el SO ni configuración de servidores.

SaaS (Software como Servicio): aplicación lista para usar entregada por Internet. El proveedor gestiona todo (infraestructura, plataforma, aplicación). El cliente no tiene control sobre la configuración subyacente, solo usa la funcionalidad. Menor control, mayor abstracción.
